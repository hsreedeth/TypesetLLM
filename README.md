## **Typeset LLM generated notes/reports (or GitHub flavored markdown (GFM))**

Minimal, practical tools to convert Markdown (often generated by LLMs, often GFM) into publication-quality PDFs using Pandoc + XeLaTeX.

This repository provides a small command-line converter and an HTTP API wrapper that produce PDFs from Markdown using a LaTeX template and a set of Pandoc filters.

The README below focuses on how to get the project running, how to use the CLI and API, and where to look if something fails.

## What this repo contains

- `src/cli.py` — a simple `md2pdf` style CLI that calls Pandoc (via `pypandoc`) with a LaTeX template and Lua filters.
- `src/api.py` — a small FastAPI wrapper that accepts Markdown and returns a generated PDF (includes basic rate-limiting and input sanitization).
- `templates/` — LaTeX templates (the default is `vintage.tex`).
- `filters/` — Pandoc Lua filters used during conversion (column/layout helpers, width adjustments, etc.).
- `fonts/`, `assets/` — example fonts and static assets referenced by the template.
- `tests/` — minimal tests for CLI and API behaviors.

## Requirements

- Python 3.8+
- Pandoc installed and on `PATH` (https://pandoc.org)
- XeLaTeX (part of a TeX distribution such as TeX Live or MiKTeX)
- `pip install -r requirements.txt`

On macOS you can install the essentials via Homebrew:

```bash
brew install pandoc
brew install --cask mactex  # or install TeX Live manually
python3 -m pip install -r requirements.txt
```

If you prefer a Linux container, install `pandoc` and a TeX distribution in that environment.

## CLI: `md2pdf` (via `src/cli.py`)

Basic usage (from the repo root):

```bash
python -m src.cli path/to/source.md --markdown-format gfm
```

Options:
- `--output, -o` — path to the output PDF (defaults to `<source>.pdf`).
- `--markdown-format` — `gfm` (default) or `commonmark_x`.

What it does:
- Uses the `templates/vintage.tex` template by default.
- Runs Pandoc with `--pdf-engine=xelatex` and the configured Lua filter(s).

Common failure modes and fixes:
- Pandoc not found: ensure `pandoc` is on your `PATH`.
- XeLaTeX missing packages: install a full TeX distribution or the missing package shown in the Pandoc/XeLaTeX error output.

## API: `src/api.py`

The API exposes a `/convert` POST endpoint that accepts JSON with `text` and optional `theme`:

Example request body:

```json
{ "text": "# My doc\nSome markdown content", "theme": "vintage" }
```

Notes about the API implementation:
- Basic rate limiting is enabled (`slowapi`) to prevent abuse (60/hour per IP).
- Input is sanitized to remove dangerous LaTeX commands (basic regex-based stripping).
- The API writes temporary files and returns the PDF as a file response; temporary artifacts are scheduled for cleanup.

To run locally for testing:

```bash
uvicorn src.api:app --reload
curl -X POST "http://127.0.0.1:8000/convert" -H "Content-Type: application/json" -d '{"text":"# hi","theme":"vintage"}' --output out.pdf
```

## Templates and Filters

- Templates are in `templates/`. The default `vintage.tex` is used if the requested theme is missing.
- Filters in `filters/` are Pandoc Lua filters; they perform layout adjustments (column widths, landscape handling, etc.).

If you add or modify a template or filter, test locally with the CLI before relying on the API.

## Tests

There are minimal tests under `tests/` (`test_cli.py`, `test_api.py`). They are lightweight and intended to validate basic behavior rather than full rendering.

Run tests with `pytest`:

```bash
python -m pip install -r requirements.txt
pytest -q
```

## Troubleshooting checklist

- Confirm `pandoc` and `xelatex` are installed and available in shell sessions the Python process inherits.
- If Pandoc conversion fails, run the same `pypandoc` command manually (or run `pandoc` directly) to see the LaTeX output and error messages.
- Check `templates/vintage.tex` for font/package references that may require additional TeX packages.
